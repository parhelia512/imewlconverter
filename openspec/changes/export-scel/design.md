## 上下文

项目已有完整的 scel 文件解析逻辑（`SougouPinyinScel.cs`），通过逆向工程了解了 scel 二进制格式的结构。当前类只实现了 `IWordLibraryImport` 接口，需要新增 `IWordLibraryExport` 的实现来完成双向转换。

scel 文件是搜狗输入法专有的二进制细胞词库格式，结构如下：
- 文件头（magic number + 保留区域）：0x0000 - 0x011F
- 统计信息区：0x0120 - 0x012F（词组数、词条总数）
- 元信息区：0x0130 - 0x153F（名称、类型、描述、示例词，UTF-16LE 编码）
- 拼音表：0x1540 起（Int32 条目数 + 逐条拼音索引）
- 词条数据区：紧跟拼音表（按同音词组组织）

所有整数为小端序，所有文本为 UTF-16LE。

## 目标 / 非目标

**目标：**
- 实现 `IWordLibraryExport` 接口，将 `WordLibraryList` 序列化为合法的 scel 二进制文件
- 生成的 scel 文件可被搜狗输入法正常识别和导入
- 在 CLI 中支持 `-o scel` 参数
- 保持与现有导出架构的一致性

**非目标：**
- 不实现 scel 文件的元信息自定义（名称、类型、描述等使用默认值即可）
- 不解析 unknown 字段的具体含义（使用已知的安全默认值）
- 不支持搜狗词库的加密格式
- 不支持 .scel 到 .scel 的元信息保留（转换后为新文件）

## 决策

### 1. 导出策略：新建独立导出类 vs 在现有类上新增接口

**选择**：在现有 `SougouPinyinScel` 类上新增 `IWordLibraryExport` 实现。

**理由**：
- 项目中其他格式（如 `QQPinyin`、`GooglePinyin`、`Rime`）均在同一个类上同时实现 Import 和 Export
- 导出逻辑需要复用已有的拼音表结构和格式常量
- 保持架构一致性

### 2. 二进制文件写入方式：`IWordLibraryExport.Export()` 返回字符串 vs 二进制流

**选择**：重写 `Export()` 方法，在方法内部生成二进制文件并写入临时文件，返回文件路径字符串列表。同时需要实现一个额外的二进制导出机制。

**实际方案**：由于 `IWordLibraryExport.Export()` 返回 `IList<string>`（文本内容），而 scel 是二进制格式，需要引入 `IWordLibraryBinExport` 接口或在 `MainBody.ExportToFile()` 中增加对二进制导出的特殊处理。

参考现有代码中 `MainBody.ExportToFile()` 的逻辑，它会将 `ExportContents` 作为文本写入文件。对于二进制格式，最佳方案是：
- 新增 `IBinaryWordLibraryExport` 接口，提供 `ExportToBinary(WordLibraryList, string outputPath)` 方法
- 在 `MainBody` 中检测导出目标是否实现了此接口，优先使用二进制导出路径

**替代方案考虑**：
- 将二进制内容 Base64 编码后放入 `Export()` 返回值 → 不优雅，破坏语义
- 在 `ExportLine()` 中抛出异常 → 与 Import 端的 `ImportLine()` 做法一致，可接受

### 3. 拼音表构建策略

**选择**：从输入的 `WordLibraryList` 中收集所有不重复的拼音音节，按字母序排序后生成拼音索引表。

**理由**：
- 搜狗原始 scel 文件的拼音表包含约 413 个标准汉语拼音音节
- 可以使用项目中已有的 `PinyinHelper` 资源获取完整拼音表，确保兼容性
- 按字母序排列与原始文件行为一致

### 4. unknown 字段的默认值

**选择**：
- unknown1（2字节）：固定写入 `10`（所有观察样本中此值恒为10）
- unknown2（4字节）：写入 `0`（无法确定其含义，0 为安全默认值）
- extra（6字节）：全部写入 `0`

**理由**：基于对大量 scel 文件的分析，这些值不影响搜狗输入法的导入功能。搜狗输入法在导入时主要校验文件头 magic number、拼音表完整性和词条数据格式。

### 5. 文件头 magic number

**选择**：使用固定文件头 `40 15 00 00 44 43 53 01`，其余保留区域填充 0。

**理由**：所有合法 scel 文件均以此 8 字节开头，搜狗输入法通过这些字节识别文件格式。

### 6. 词条数据组织

**选择**：将具有相同拼音序列的词条合并为同音词组，按拼音序排列写入。

**理由**：
- scel 格式要求同音词必须归到同一个词组中
- 按拼音序排列有利于搜狗输入法内部的索引查找
- 0x0120 处的 dictLen 值为词组数（不展开同音词），需正确计算

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| unknown 字段值不正确导致搜狗输入法拒绝导入 | 使用已验证的默认值（10, 0, 全零）；可通过实际导入搜狗输入法进行端到端验证 |
| 拼音表不完整导致部分词条无法正确映射 | 使用完整的标准拼音音节表（来自 PinyinHelper 资源），覆盖所有合法拼音 |
| 生成文件过大超出搜狗输入法限制 | 观察到搜狗官方词库单文件可达数万词条，暂不限制；后续可按需分割 |
| `IWordLibraryExport` 接口设计不适合二进制格式 | 新增 `IBinaryWordLibraryExport` 接口，并在 `MainBody` 中增加对应的调用路径，不破坏现有文本导出流程 |
| 词条中的拼音信息不完整或不准确（来源词库无拼音） | 在导出前依赖现有的 `PinyinGenerater` 进行拼音生成，确保每个词条都有拼音 |
