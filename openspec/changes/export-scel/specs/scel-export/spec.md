## 新增需求

### 需求:导出词库为 scel 二进制格式

系统必须支持将内部 `WordLibraryList` 导出为搜狗细胞词库 `.scel` 二进制文件格式。导出的文件必须符合搜狗输入法的 scel 文件规范，能够被搜狗输入法正常识别和导入。

#### 场景:基本词库导出

- **当** 用户指定导出格式为 scel，并提供包含拼音和汉字的词库数据
- **那么** 系统生成一个合法的 `.scel` 二进制文件，文件头以 `40 15 00 00 44 43 53 01` 开头

#### 场景:导出文件可被搜狗输入法导入

- **当** 系统生成的 scel 文件被用户导入搜狗输入法
- **那么** 搜狗输入法必须能正常识别该文件并成功导入所有词条

#### 场景:包含多个同音词的词库导出

- **当** 词库中包含多个拼音相同的词条（如 "世界" 和 "实际" 都是 shi'jie）
- **那么** 系统必须将同音词合并到同一个词组中写入，词组头部的 same_py_count 值正确反映同音词数量

### 需求:生成合法的 scel 文件头

系统生成的 scel 文件必须包含完整的文件头结构，包括 magic number、统计信息和元信息区域。

#### 场景:文件头 magic number 正确

- **当** 系统写入 scel 文件
- **那么** 文件偏移 0x0000 处必须写入 8 字节固定值 `40 15 00 00 44 43 53 01`

#### 场景:统计信息正确

- **当** 系统写入包含 N 个词组（同音词归并后）和 M 个词条（展开后）的词库
- **那么** 偏移 0x0120 处必须写入词组数 N（Int32 LE），偏移 0x0124 处必须写入词条总数 M（Int32 LE）

#### 场景:元信息区域完整

- **当** 系统写入 scel 文件
- **那么** 偏移 0x0130（名称）、0x0338（类型）、0x0540（描述）、0x0D40（示例词）处必须写入 UTF-16LE 编码的字符串并以 \0 结尾，区域长度分别为 520、520、2048、2048 字节

### 需求:生成正确的拼音索引表

系统必须从词库数据中构建完整的拼音索引表，写入到文件偏移 0x1540 处。

#### 场景:拼音表格式正确

- **当** 系统写入拼音表
- **那么** 偏移 0x1540 处必须先写入拼音条目总数（Int32 LE），然后逐条写入每个拼音的索引号（Int16 LE）、字节长度（Int16 LE）和 UTF-16LE 编码的拼音字符串

#### 场景:拼音表覆盖所有词条

- **当** 词库中的词条包含各种拼音音节
- **那么** 拼音表必须包含所有词条中出现的拼音音节，每个拼音只出现一次，索引从 0 开始递增

### 需求:正确写入词条数据区

系统必须按同音词组格式写入所有词条数据，紧跟在拼音表之后。

#### 场景:词条数据格式正确

- **当** 系统写入一个词条
- **那么** 必须按以下顺序写入：汉字字节数（UInt16 LE）、汉字内容（UTF-16LE）、unknown1（Int16 LE，值为 10）、unknown2（Int32 LE，值为 0）、extra（6 字节全零）

#### 场景:词组结构完整

- **当** 系统写入一个词组（包含 N 个同音词，共享 M 个拼音音节）
- **那么** 必须先写入词组头部：same_py_count=N（UInt16 LE）和 pinyin_byte_len=M*2（UInt16 LE），然后写入拼音索引数组（M 个 UInt16 LE），最后依次写入 N 个词条

### 需求:CLI 支持 scel 导出

命令行工具必须支持通过 `-o scel` 参数指定导出为搜狗细胞词库格式。

#### 场景:CLI 导出为 scel

- **当** 用户执行命令 `ImeWlConverterCmd -i ggpy -o scel -O output.scel input.txt`
- **那么** 系统必须读取 Google 拼音格式的输入文件，将词库转换并导出为 scel 格式的 output.scel 文件

#### 场景:导出文件扩展名

- **当** 用户指定导出格式为 scel 且未指定输出文件扩展名
- **那么** 系统必须使用 `.scel` 作为输出文件的默认扩展名

### 需求:词条拼音完整性保证

导出时必须确保每个词条都包含完整的拼音信息。

#### 场景:无拼音词条的处理

- **当** 词库中存在没有拼音信息的词条
- **那么** 系统必须在导出前使用 PinyinGenerater 为其生成拼音，若无法生成则跳过该词条并报告错误

#### 场景:拼音格式标准化

- **当** 词条的拼音包含非标准格式（如声调数字、大写字母）
- **那么** 系统必须将拼音转换为小写无声调的标准格式后再写入拼音索引表
